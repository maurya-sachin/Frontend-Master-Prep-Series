### By Experience Level

**Junior (0-2 years)**
1. Start: JavaScript Fundamentals (01-javascript)
2. Then: React Basics (03-react/01-04)
3. Then: HTML/CSS (05-html-css)

5. Build: 2-3 simple projects

1. Start: React Advanced (03-react/05-08)
2. Then: Next.js (04-nextjs)
3. Then: Performance & Testing (06-testing, 07-performance)

1. Start: Architecture (09-architecture)
2. Then: System Design patterns
3. Then: Security & Performance (10-security, 07-performance)
4. Practice: Hard problems + system design

**Coding Round**
- Focus: 18-coding-challenges (all categories)
- Master: JavaScript fundamentals, algorithms
- Practice: LeetCode Medium/Hard
- Time: 2 weeks minimum

**System Design**
- Focus: 09-architecture + real-world examples
- Master: Scalability, trade-offs, design patterns
- Practice: Design Twitter, YouTube, Autocomplete
- Time: 1 week minimum

**Take-Home Project**
- Focus: Full-stack implementation
- Master: Clean code, testing, documentation
- Build: From scratch with best practices
- Time: 1-2 days for project

## üéì Study Techniques

### 1. Active Recall (Most Effective)
- Use flashcards daily
- Don't just read - quiz yourself
- Explain concepts out loud
- **Time:** 15-30 min daily

### 2. Spaced Repetition
- Review on days: 1, 3, 7, 14, 30
- Focus more time on difficult cards
- Use our flashcard system
- **Time:** 15-20 min daily

### 3. Feynman Technique
- Pick a concept
- Explain it to a 5-year-old (or rubber duck)
- Identify gaps in understanding
- Simplify and repeat
- **Time:** 10-15 min per concept

### 4. Pomodoro Technique
- Study for 25 minutes
- Break for 5 minutes
- After 4 pomodoros, take 15-30 min break
- Prevents burnout
- **Time:** Entire study session

### 5. Project-Based Learning
- Build real projects
- Apply concepts immediately
- Learn by doing
- **Time:** Weekends (4-8 hours)

## üí° Pro Tips

1. **Don't Skip Fundamentals**
   - Even seniors should review basics
   - Interviewers love asking "simple" questions

2. **Code Every Day**
   - Even 30 minutes matters
   - Consistency > Intensity

3. **Mock Interviews are Critical**
   - Practice talking while coding
   - Get comfortable with pressure

4. **Understand, Don't Memorize**
   - Explain the "why" not just "how"
   - Interviewers ask follow-ups

5. **Track Weak Areas**
   - Double down on difficult topics
   - Don't avoid them

6. **Stay Healthy**
   - Sleep well
   - Exercise
   - Take breaks
