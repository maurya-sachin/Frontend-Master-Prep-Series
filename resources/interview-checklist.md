# Interview Day Checklist

> **Step-by-step checklist to ensure you're fully prepared for your frontend interview**

---

## üìÖ 1 Week Before Interview

### Technical Preparation
- [ ] Review all weak flashcard topics
- [ ] Complete 20+ coding problems similar to company's style
- [ ] Do 2-3 full mock interviews
- [ ] Review company's tech stack (React, Next.js, etc.)
- [ ] Prepare system design examples (2-3 you can explain well)

### Research Company
- [ ] Read company's engineering blog
- [ ] Check recent product launches
- [ ] Understand company's mission and values
- [ ] Review Glassdoor interview experiences
- [ ] Research interviewer's background on LinkedIn

### Logistics
- [ ] Test video call setup (camera, mic, lighting)
- [ ] Prepare quiet interview space
- [ ] Check internet connection stability
- [ ] Have backup plan (mobile hotspot, different room)

---

## üìÖ 1 Day Before Interview

### Final Review (2-3 hours max)
- [ ] Review [Pre-Interview 30 Flashcards](../19-flashcards/curated-decks/pre-interview-30.md) (15 min)
- [ ] Quick scan of [Essential 50](../19-flashcards/curated-decks/essential-50.md) weak cards (20 min)
- [ ] Review 3-5 coding problems you struggled with (45 min)
- [ ] Practice explaining 1 project out loud (15 min)
- [ ] Review system design principles (30 min)

### Technical Setup
- [ ] Update IDE/code editor
- [ ] Test screen sharing
- [ ] Prepare code snippets (if take-home)
- [ ] Charge laptop fully
- [ ] Close unnecessary apps/tabs

### Materials Ready
- [ ] Resume (latest version)
- [ ] Notepad and pen
- [ ] Water bottle
- [ ] Questions to ask interviewer (prepare 5-7)
- [ ] Past project code (in case of discussion)

### Mental Prep
- [ ] **Get 7-8 hours of sleep** (most important!)
- [ ] Light exercise (reduce stress)
- [ ] Avoid cramming (causes anxiety)
- [ ] Review your success stories
- [ ] Visualize successful interview

---

## üìÖ Morning of Interview

### Morning Routine (1-2 hours before)
- [ ] Wake up 2-3 hours before interview
- [ ] Healthy breakfast (avoid heavy meals)
- [ ] Light exercise (10-15 min walk/stretch)
- [ ] Shower and dress professionally (even for remote)
- [ ] Review [Daily 20 Flashcards](../19-flashcards/curated-decks/daily-20.md) (10 min)

### Final Technical Check (45 min before)
- [ ] Test camera, microphone, speakers
- [ ] Test screen sharing
- [ ] Close all unnecessary tabs/apps
- [ ] Silence phone notifications
- [ ] Open code editor, browser, notepad
- [ ] Have interview link ready

### Mental Prep (30 min before)
- [ ] **Review Pre-Interview 30 one last time** (15 min)
- [ ] Deep breathing exercises (5 min)
- [ ] Review your "Why I want this job" notes
- [ ] Remind yourself of your strengths
- [ ] **Smile** - it helps with confidence!

---

## üìÖ 30 Minutes Before Interview

### Final Checks
- [ ] Use restroom
- [ ] Pour water (keep nearby)
- [ ] Put notepad and pen ready
- [ ] Set phone to Do Not Disturb
- [ ] Close door, hang "Do Not Disturb" sign

### Mental State
- [ ] **Deep breaths** - 4 seconds in, 4 seconds hold, 4 seconds out
- [ ] Positive affirmations: "I am prepared. I've got this."
- [ ] Review quick wins from practice
- [ ] Remember: It's a conversation, not an interrogation

### Technical Quick Review (10 min)
- [ ] Skim these key concepts:
  - JavaScript: Closures, event loop, promises
  - React: Hooks, Virtual DOM, performance
  - Your recent project (elevator pitch ready)

---

## üìÖ During Interview

### Opening (First 5 min)
- [ ] Greet warmly, smile
- [ ] Brief introduction (30 seconds)
- [ ] Thank them for their time
- [ ] Ask if they can see/hear you clearly

### Coding Round
- [ ] **Read problem 2-3 times** before coding
- [ ] Ask clarifying questions
- [ ] Explain approach before coding
- [ ] Think out loud while coding
- [ ] Start with brute force, then optimize
- [ ] Write test cases
- [ ] Analyze time/space complexity
- [ ] Handle edge cases

### System Design Round
- [ ] Clarify requirements (5-10 min)
- [ ] Define scope (what's in/out)
- [ ] Start with high-level diagram
- [ ] Discuss trade-offs
- [ ] Think about scalability
- [ ] Consider edge cases
- [ ] Ask for feedback throughout

### Behavioral Questions
- [ ] Use STAR method (Situation, Task, Action, Result)
- [ ] Be specific with examples
- [ ] Highlight your impact with numbers
- [ ] Be honest about challenges
- [ ] Show what you learned

### Throughout Interview
- [ ] Maintain eye contact (look at camera)
- [ ] Listen carefully, don't interrupt
- [ ] It's okay to pause and think
- [ ] If stuck, explain your thought process
- [ ] Ask for hints if truly stuck (after trying)
- [ ] Stay positive, even if struggling

---

## üìÖ Closing (Last 5-10 min)

### Your Questions to Ask
Have 5-7 questions ready. Examples:

**About the Role**
- [ ] "What does success look like in this role after 6 months?"
- [ ] "What are the biggest technical challenges the team is facing?"
- [ ] "How is the frontend team structured?"

**About the Team**
- [ ] "What's the code review process like?"
- [ ] "How does the team handle technical debt?"
- [ ] "What's the deployment frequency?"

**About Growth**
- [ ] "What learning and development opportunities are available?"
- [ ] "How do you approach mentorship?"

**Next Steps**
- [ ] "What are the next steps in the interview process?"
- [ ] "When can I expect to hear back?"
- [ ] "Is there anything else I can provide or clarify?"

### Before Leaving
- [ ] Thank interviewer for their time
- [ ] Express genuine interest in role
- [ ] Mention something specific you learned/enjoyed
- [ ] Confirm next steps
- [ ] Ask for interviewer's email (for follow-up)

---

## üìÖ After Interview

### Immediate (Within 1 hour)
- [ ] Write down interview questions (while fresh)
- [ ] Note what went well
- [ ] Note what could be improved
- [ ] Send thank you email (within 24 hours)

### Thank You Email Template
```
Subject: Thank you - [Your Name] - [Position] Interview

Hi [Interviewer Name],

Thank you for taking the time to speak with me about the [Position] role today.

I particularly enjoyed our discussion about [specific topic you discussed].
It reinforced my excitement about [something specific about the role/company].

I'm very interested in contributing to [specific project/goal mentioned],
and I believe my experience with [relevant skill] would add value to the team.

Please don't hesitate to reach out if you need any additional information.

Looking forward to hearing from you about next steps!

Best regards,
[Your Name]
```

### Follow-up (If no response after timeline they mentioned)
- [ ] Wait until their mentioned timeline + 2 days
- [ ] Send polite follow-up email
- [ ] Express continued interest
- [ ] Ask for any updates
