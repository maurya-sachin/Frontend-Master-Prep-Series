**Priority Topics:**
1. Advanced React (custom hooks, performance)

- Scale considerations
- Team collaboration strategies

**Recommended Problems:**
- All problems (know multiple solutions)

- Design component library architecture

## üí° Study Tips

### Active Recall (Most Effective)

1. **Flashcards Daily** (15 min/day)
   - Morning: New cards
   - Evening: Review cards
   - Use spaced repetition

2. **Code Without Looking** (30 min/day)
   - Pick a problem
   - Code from memory
   - Check solution only after attempt

3. **Teach Someone** (Weekly)
   - Explain concepts out loud
   - Blog about what you learned
   - Pair program with friend

### Spaced Repetition Schedule

| Day | Review |
|-----|--------|
| 0 | Learn new concept |
| 1 | Review (after 1 day) |
| 3 | Review (after 3 days) |
| 7 | Review (after 1 week) |
| 14 | Review (after 2 weeks) |
| 30 | Review (after 1 month) |

### Mock Interview Tips

1. **Practice Out Loud**
   - Don't just think, speak
   - Explain your thought process
   - Ask clarifying questions

2. **Time Yourself**
   - 45 minutes per coding problem
   - 20 minutes per system design
   - 5 minutes per concept question

3. **Record Yourself** (Optional)
   - Watch for nervous habits
   - Check explanation clarity
   - Improve communication

## üìû During the Interview

### Problem-Solving Process

1. **Listen Carefully** (2 min)
   - Don't interrupt
   - Take notes
   - Wait for complete question

2. **Clarify Requirements** (3 min)
   - Ask about edge cases
   - Confirm input/output format
   - Check constraints

3. **Explain Approach** (5 min)
   - Think out loud
   - Discuss trade-offs
   - Get interviewer buy-in

4. **Code Solution** (25 min)
   - Start simple, optimize later
   - Talk while coding
   - Use clear variable names

5. **Test Your Code** (10 min)
   - Walk through examples
   - Check edge cases
   - Fix bugs

6. **Optimize** (5 min)
   - Discuss time/space complexity
   - Suggest improvements
   - Alternative approaches

### Communication Tips

- **"I'm thinking about..."** (shows thought process)
- **"Let me clarify..."** (asks good questions)
- **"One approach could be..."** (discusses options)
- **"The trade-off here is..."** (shows decision-making)
- **"I would test this by..."** (shows thoroughness)
